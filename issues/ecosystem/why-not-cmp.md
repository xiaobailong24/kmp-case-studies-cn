# Why not Compose Multiplatform

> Issue [#38](https://github.com/xiaobailong24/kmp-case-studies-cn/issues/38) · 标签：`ecosystems`

## 概述

探讨 Compose Multiplatform 的局限性和不适用场景，帮助开发者做出更客观的技术选型。

### 可能不适合的场景

- 团队主要是 iOS 开发者，无 Kotlin 经验
- 对 iOS 原生体验有极致要求
- 需要大量使用平台特定 API
- 项目生命周期短，无需长期维护

### 需要权衡的因素

- iOS 生态工具链成熟度
- 调试体验（尤其是 iOS 端）
- 团队学习曲线
- 第三方库跨平台支持程度
